# Honcho API Reference

> API version: **v3** · Self-hosted FastAPI server · [Honcho GitHub](https://github.com/plastic-labs/honcho)

## 1. Overview

| Item         | Value                                                   |
| ------------ | ------------------------------------------------------- |
| Base URL     | `http://localhost:8000` (configurable via `/v3` prefix) |
| Auth         | None — targets trusted network, no authentication layer |
| OpenAPI spec | `GET /openapi.json` (auto-generated by FastAPI)         |
| Swagger UI   | `/docs` on running instance                             |
| ReDoc        | `/redoc` on running instance                            |

Honcho-Panel auto-detects the API URL at runtime (see [DEPLOYMENT.md](DEPLOYMENT.md)).

## 2. Data Model

```
Workspace                          ← top-level container
├── Peer                           ← any participant (human or AI agent)
│   └── (internal collections)     ← vector-embedded observations, keyed by (observer, observed)
├── Session                        ← conversation context (many-to-many with peers)
│   ├── Peer (participant)
│   └── Message                    ← atomic user/agent utterance
└── Conclusion                     ← extracted insight about a peer
```

- **Workspace**: isolates data between use cases (formerly called "App").
- **Peer**: first-class entity — humans and AI agents alike.
- **Session**: a thread/conversation involving one or more peers.
- **Message**: labelled by source peer; lives on a session.
- **Conclusion**: a deductive/inductive insight extracted from messages by the background deriver.
- **Representation**: a static snapshot of what Honcho knows about a peer (session-scoped).
- **Peer Card**: a compact identity summary (list of tags/traits).

Background processing (deriver) runs asynchronously. Newly-added messages may take a moment to be reflected in representations and chat responses.

## 3. Health

### `GET /health`

Check if the Honcho server is reachable.

**Response** `200`:

```json
{ "status": "ok" }
```

> **Panel usage:** Settings page calls this to test the connection before saving the API URL.

***

## 4. Workspaces

### `POST /workspaces`

Get or create a workspace. Returns `201` if created, `200` if already exists.

**Request:**

```json
{ "name": "my-ai-agent" }
```

**Response:**

```json
{
  "id": "ws_abc123",
  "name": "my-ai-agent",
  "created_at": "2025-01-15T10:30:00Z",
  "config": {}
}
```

**Panel usage:** Workspace selector dropdown.

***

### `POST /workspaces/list`

List all workspaces (paginated).

**Request:**

```json
{
  "page": 1,
  "page_size": 25
}
```

**Response:**

```json
{
  "items": [
    { "id": "ws_abc123", "name": "my-ai-agent", "created_at": "..." },
    { "id": "ws_def456", "name": "customer-bot", "created_at": "..." }
  ],
  "total": 2,
  "page": 1,
  "page_size": 25
}
```

**Panel usage:** Populates the workspace dropdown and workspace list.

***

### `GET /workspaces/{workspace_id}/queue/status`

Background processing queue status for a workspace.

**Response:**

```json
{
  "pending": 3,
  "processing": 1,
  "completed": 245,
  "failed": 0
}
```

**Panel usage:** Overview page — "Queue Length" stat card.

***

### `DELETE /workspaces/{workspace_id}`

Delete a workspace (asynchronous, returns `202`). Removes all peers, sessions, messages, and conclusions within it.

***

## 5. Peers

### `POST /peers/list`

List peers in a workspace (paginated).

**Request:**

```json
{
  "page": 1,
  "page_size": 50
}
```

**Response:**

```json
{
  "items": [
    { "id": "peer_123", "name": "user-123", "created_at": "..." },
    { "id": "peer_456", "name": "developer-kai", "created_at": "..." }
  ],
  "total": 12,
  "page": 1,
  "page_size": 50
}
```

**Panel usage:** Peers page table.

***

### `POST /peers`

Get or create a peer. Returns `201` if created, `200` if already exists.

**Request:**

```json
{ "name": "user-123" }
```

***

### `POST /peers/{peer_id}/chat`

Natural-language query over what Honcho knows about a peer. Returns a reasoning-grounded text response.

**Request:**

```json
{
  "query": "What learning styles does the user prefer?",
  "session_id": "session-1",
  "reasoning_level": "medium",
  "stream": false
}
```

**Response:**

```json
{
  "content": "Based on previous interactions, the user responds best to visual explanations with concrete examples. They tend to ask follow-up questions when concepts are presented text-only."
}
```

**Reasoning levels:** `low`, `medium`, `high`, `max`.

**Panel usage:** Peer Detail page → Chat tab.

***

### `GET /peers/{peer_id}/card`

List of stable biographical facts about a peer. Each entry is a full sentence.

**Response:**
```json
{
  "peer_card": [
    "Name: Alice. Also known as 'Ali'.",
    "PREFERENCE: Prefers casual tone.",
    "INSTRUCTION: Never mention her ex-boyfriend.",
    "TRAIT: Detail-oriented, prefers data over anecdotes."
  ]
}
```

**Panel usage:** Peer Detail page → Card tab.

***

### `POST /peers/{peer_id}/representation`

Static representation document — a snapshot of conclusions about a peer.

**Request:**

```json
{
  "session_id": "session-1",
  "include_most_frequent": true,
  "max_conclusions": 20
}
```

**Response:**

```json
{
  "representation": "# user-123\n\n- Prefers visual explanations over text\n- Works primarily in TypeScript and React\n- Most active between 9 AM and noon\n..."
}
```

**Panel usage:** Peer Detail page → Representation tab.

***

## 6. Sessions

### `POST /sessions/list`

List sessions in a workspace (paginated).

**Request:**

```json
{
  "page": 1,
  "page_size": 50,
  "peer_id": "peer_123"
}
```

**Response:**

```json
{
  "items": [
    { "id": "ses_abc123", "name": "session-1", "created_at": "..." },
    { "id": "ses_def456", "name": "session-2", "created_at": "..." }
  ],
  "total": 24,
  "page": 1,
  "page_size": 50
}
```

**Panel usage:** Sessions page table.

***

### `POST /sessions`

Get or create a session. Optionally attaches peers on creation.

**Request:**

```json
{
  "name": "session-1",
  "peers": {
    "user-123": { "observation_enabled": true },
    "assistant": { "observation_enabled": true }
  }
}
```

**Panel usage:** Implicit — sessions are created when peers first interact.

***

### `DELETE /sessions/{session_id}`

Delete a session and all its messages (asynchronous, returns `202`).

**Panel usage:** Session list — delete action.

***

### `GET /sessions/{session_id}/context`

Prompt-ready context bundle: messages + summary + peer representation + peer card.

**Query parameters:**

| Param                     | Description                         |
| ------------------------- | ----------------------------------- |
| `tokens` *int*            | Max token budget for the response   |
| `summary` *bool*          | Include session summary             |
| `peer_target` *str*       | Target peer for representation      |
| `search_query` *str*      | Hybrid search query                 |
| `search_top_k` *int*      | Search results to include           |
| `max_conclusions` *int*   | Max conclusions                     |
| `limit_to_session` *bool* | Only use messages from this session |

**Response:**

```json
{
  "name": "session-1",
  "messages": [
    { "id": "msg_1", "content": "Hello!", "peer_name": "user-123", "created_at": "..." },
    { "id": "msg_2", "content": "Hi! How can I help?", "peer_name": "assistant", "created_at": "..." }
  ],
  "summary": {
    "short": "A greeting exchange.",
    "long": "user-123 initiated a conversation. The assistant responded helpfully.",
    "token_count": 45
  },
  "peer_representation": "# user-123\n...",
  "peer_card": ["Name: user-123"]
}
```

**Panel usage:** Session View page — loads all messages and context.

***

## 7. Messages

### `POST /messages`

Batch create one or more messages on a session.

**Request:**

```json
{
  "messages": [
    {
      "content": "Hey there — can you help me with my math homework?",
      "peer_id": "user-123"
    },
    {
      "content": "Absolutely. Send me your first problem!",
      "peer_id": "assistant"
    }
  ]
}
```

**Response:**

```json
{
  "messages": [
    {
      "id": "msg_abc",
      "public_id": "msg_abc",
      "content": "Hey there — can you help me with my math homework?",
      "peer_name": "user-123",
      "session_name": "session-1",
      "created_at": "2025-05-14T14:30:22Z",
      "metadata": {},
      "configuration": null
    }
  ]
}
```

**Panel usage:** Peer Detail Chat — sending a message.

***

### `POST /messages/list`

List messages (paginated, with optional filters).

**Request:**

```json
{
  "page": 1,
  "page_size": 100,
  "peer_id": "user-123"
}
```

**Response:**

```json
{
  "items": [
    { "id": "msg_1", "content": "...", "peer_name": "user-123", "created_at": "..." },
    { "id": "msg_2", "content": "...", "peer_name": "assistant", "created_at": "..." }
  ],
  "total": 18,
  "page": 1,
  "page_size": 100
}
```

**Panel usage:** Session View — message timeline

***

## 8. Conclusions

### `POST /conclusions/list`

List conclusions (paginated, filterable).

**Request:**

```json
{
  "page": 1,
  "page_size": 25,
  "filters": {
    "observed": "user-123"
  }
}
```

**Response:**

```json
{
  "items": [
    {
      "id": "con_abc",
      "content": "The user prefers visual explanations over text",
      "observer": "honcho",
      "observed": "user-123",
      "confidence": 0.92,
      "created_at": "2025-05-14T10:00:00Z"
    }
  ],
  "total": 15,
  "page": 1,
  "page_size": 25
}
```

**Panel usage:** Conclusions page list.

***

### `POST /conclusions/query`

Semantic search over conclusions. Returns results ranked by relevance.

**Request:**

```json
{
  "query": "learning preferences",
  "top_k": 10,
  "distance": 0.5,
  "filters": {
    "observed": "user-123"
  }
}
```

**Response:**

```json
{
  "items": [
    {
      "id": "con_abc",
      "content": "The user prefers visual explanations over text",
      "observer": "honcho",
      "observed": "user-123",
      "confidence": 0.92
    }
  ]
}
```

**Panel usage:** Conclusions page — semantic search bar.

***

### `DELETE /conclusions/{conclusion_id}`

Delete a conclusion. Returns `204`.

**Panel usage:** Conclusions page — delete action.

***

## 9. Unused Endpoints (Future)

These endpoints are available but not currently used by Honcho-Panel.

### Workspaces

| Method | Path                                        | Description                                                 |
| ------ | ------------------------------------------- | ----------------------------------------------------------- |
| `PUT`  | `/workspaces/{workspace_id}`                | Update workspace metadata/config                            |
| `POST` | `/workspaces/{workspace_id}/search`         | Hybrid search (BM25 + vector) across all workspace messages |
| `POST` | `/workspaces/{workspace_id}/schedule_dream` | Manually schedule a dream task                              |

### Peers

| Method | Path                        | Description                                   |
| ------ | --------------------------- | --------------------------------------------- |
| `PUT`  | `/peers/{peer_id}`          | Update peer metadata/config                   |
| `POST` | `/peers/{peer_id}/sessions` | List sessions for a specific peer (paginated) |
| `GET`  | `/peers/{peer_id}/context`  | Combined representation + card                |
| `POST` | `/peers/{peer_id}/search`   | Search messages by/from this peer             |

### Sessions

| Method   | Path                                            | Description                                      |
| -------- | ----------------------------------------------- | ------------------------------------------------ |
| `PUT`    | `/sessions/{session_id}`                        | Update session metadata/config                   |
| `POST`   | `/sessions/{session_id}/clone`                  | Clone a session (optionally up to a message\_id) |
| `POST`   | `/sessions/{session_id}/peers`                  | Add peers to session                             |
| `PUT`    | `/sessions/{session_id}/peers`                  | Replace all peers in session                     |
| `DELETE` | `/sessions/{session_id}/peers`                  | Remove peers from session                        |
| `GET`    | `/sessions/{session_id}/peers`                  | List peers in session (paginated)                |
| `GET`    | `/sessions/{session_id}/peers/{peer_id}/config` | Get peer's session config                        |
| `PUT`    | `/sessions/{session_id}/peers/{peer_id}/config` | Set peer's session config                        |
| `GET`    | `/sessions/{session_id}/summaries`              | Short and long session summaries                 |
| `POST`   | `/sessions/{session_id}/search`                 | Search messages within a session                 |

### Messages

| Method | Path                     | Description                                |
| ------ | ------------------------ | ------------------------------------------ |
| `POST` | `/messages/upload`       | Upload a file → messages from text content |
| `GET`  | `/messages/{message_id}` | Get a single message by ID                 |
| `PUT`  | `/messages/{message_id}` | Update message metadata                    |

### Conclusions

| Method | Path           | Description                                       |
| ------ | -------------- | ------------------------------------------------- |
| `POST` | `/conclusions` | Batch create conclusions (admin/manual injection) |

### Webhooks

| Method   | Path                      | Description                        |
| -------- | ------------------------- | ---------------------------------- |
| `POST`   | `/webhooks`               | Get or create a webhook endpoint   |
| `GET`    | `/webhooks`               | List webhook endpoints (paginated) |
| `DELETE` | `/webhooks/{endpoint_id}` | Delete a webhook endpoint          |
| `GET`    | `/webhooks/test`          | Test publish a webhook event       |

### Metrics

| Method | Path       | Description                                    |
| ------ | ---------- | ---------------------------------------------- |
| `GET`  | `/metrics` | Prometheus metrics (if `METRICS_ENABLED=true`) |
