# Honcho API Reference

> API version: **v3** · Self-hosted FastAPI server · [Honcho GitHub](https://github.com/plastic-labs/honcho)

## 1. Overview

| Item         | Value                                                   |
| ------------ | ------------------------------------------------------- |
| Base URL     | `http://localhost:8000`                                 |
| API prefix   | `/v3` (all resource endpoints are under `/v3`)          |
| Auth         | Disabled by default (`AUTH_USE_AUTH=false`); supports HTTP Bearer token when enabled |
| OpenAPI spec | `GET /openapi.json` (auto-generated by FastAPI)         |
| Swagger UI   | `/docs` on running instance                             |
| ReDoc        | `/redoc` on running instance                            |

Honcho-Panel auto-detects the API URL at runtime (see [DEPLOYMENT.md](DEPLOYMENT.md)).

## 2. Data Model

```
Workspace                          ← top-level container
├── Peer                           ← any participant (human or AI agent)
│   └── (internal collections)     ← vector-embedded observations, keyed by (observer, observed)
├── Session                        ← conversation context (many-to-many with peers)
│   ├── Peer (participant)
│   └── Message                    ← atomic user/agent utterance
└── Conclusion                     ← extracted insight about a peer
```

- **Workspace**: isolates data between use cases (formerly called "App").
- **Peer**: first-class entity — humans and AI agents alike.
- **Session**: a thread/conversation involving one or more peers.
- **Message**: labelled by source peer; lives on a session.
- **Conclusion**: a deductive/inductive insight extracted from messages by the background deriver.
- **Representation**: a static snapshot of what Honcho knows about a peer (session-scoped).
- **Peer Card**: a compact identity summary (list of tags/traits).

Background processing (deriver) runs asynchronously. Newly-added messages may take a moment to be reflected in representations and chat responses.

## 3. Health

### `GET /health`

Check if the Honcho server is reachable.

**Response** `200`:

```json
{ "status": "ok" }
```

> **Panel usage:** Settings page calls this to test the connection before saving the API URL.

***

## 4. Workspaces

### `POST /v3/workspaces`

Get or create a workspace. If `workspace_id` is provided as a query parameter, uses that; otherwise uses the ID from the JWT. Returns `200` on success.

**Request:**

```json
{ "id": "my-ai-agent" }
```

**Response:**

```json
{
  "id": "ws_abc123",
  "created_at": "2025-01-15T10:30:00Z",
  "metadata": {},
  "configuration": {}
}
```

**Panel usage:** Workspace selector dropdown.

***

### `POST /v3/workspaces/list`

List all workspaces (paginated, with optional filters).

**Request:**

```json
{
  "filters": {}
}
```

**Query parameters:**

| Param  | Type | Default | Description   |
| ------ | ---- | ------- | ------------- |
| `page` | int  | 1       | Page number   |
| `size` | int  | 50      | Page size     |

**Response:**

```json
{
  "items": [
    { "id": "ws_abc123", "created_at": "...", "metadata": {} },
    { "id": "ws_def456", "created_at": "...", "metadata": {} }
  ],
  "total": 2,
  "page": 1,
  "size": 50,
  "pages": 1
}
```

**Panel usage:** Populates the workspace dropdown and workspace list.

***

### `GET /v3/workspaces/{workspace_id}/queue/status`

Background processing queue status for a workspace. Tracks user-facing task types (representation, summary, dream).

**Response:**

```json
{
  "total_work_units": 249,
  "completed_work_units": 245,
  "in_progress_work_units": 1,
  "pending_work_units": 3,
  "sessions": {}
}
```

**Panel usage:** Overview page — "Queue Length" stat card.

***

### `DELETE /v3/workspaces/{workspace_id}`

Delete a workspace (asynchronous, returns `202`). Removes all peers, sessions, messages, and conclusions within it.

***

## 5. Peers

### `POST /v3/workspaces/{workspace_id}/peers/list`

List peers in a workspace (paginated, with optional filters).

**Request:**

```json
{
  "filters": {}
}
```

**Query parameters:**

| Param  | Type | Default | Description   |
| ------ | ---- | ------- | ------------- |
| `page` | int  | 1       | Page number   |
| `size` | int  | 50      | Page size     |

**Response:**

```json
{
  "items": [
    { "id": "peer_123", "workspace_id": "ws_abc", "created_at": "...", "metadata": {} },
    { "id": "peer_456", "workspace_id": "ws_abc", "created_at": "...", "metadata": {} }
  ],
  "total": 12,
  "page": 1,
  "size": 50,
  "pages": 1
}
```

**Panel usage:** Peers page table.

***

### `POST /v3/workspaces/{workspace_id}/peers`

Get or create a peer. If `peer_id` is provided as a query parameter, uses that; otherwise uses the peer_id from the JWT. Returns `200` on success.

**Request:**

```json
{ "id": "user-123" }
```

***

### `POST /v3/workspaces/{workspace_id}/peers/{peer_id}/chat`

Natural-language query over what Honcho knows about a peer. Performs agentic search and reasoning.

**Request:**

```json
{
  "query": "What learning styles does the user prefer?",
  "session_id": "session-1",
  "reasoning_level": "medium",
  "stream": false,
  "target": null
}
```

**Response:**

```json
{
  "content": "Based on previous interactions, the user responds best to visual explanations with concrete examples."
}
```

**Reasoning levels:** `minimal`, `low`, `medium`, `high`, `max` (default: `low`).

**Panel usage:** Peer Detail page → Chat tab.

***

### `GET /v3/workspaces/{workspace_id}/peers/{peer_id}/card`

List of stable biographical facts about a peer from the observer's perspective.

**Response:**
```json
{
  "peer_card": [
    "Name: Alice. Also known as 'Ali'.",
    "PREFERENCE: Prefers casual tone.",
    "INSTRUCTION: Never mention her ex-boyfriend.",
    "TRAIT: Detail-oriented, prefers data over anecdotes."
  ]
}
```

**Panel usage:** Peer Detail page → Card tab.

***

### `POST /v3/workspaces/{workspace_id}/peers/{peer_id}/representation`

Static representation document — a curated subset of conclusions about a peer.

**Request:**

```json
{
  "session_id": "session-1",
  "include_most_frequent": true,
  "max_conclusions": 20
}
```

**Response:**

```json
{
  "representation": "# user-123\n\n- Prefers visual explanations over text\n- Works primarily in TypeScript and React\n- Most active between 9 AM and noon\n..."
}
```

**Panel usage:** Peer Detail page → Representation tab.

***

## 6. Sessions

### `POST /v3/workspaces/{workspace_id}/sessions/list`

List sessions in a workspace (paginated, with optional filters).

**Request:**

```json
{
  "filters": {}
}
```

**Query parameters:**

| Param  | Type | Default | Description   |
| ------ | ---- | ------- | ------------- |
| `page` | int  | 1       | Page number   |
| `size` | int  | 50      | Page size     |

**Response:**

```json
{
  "items": [
    { "id": "ses_abc123", "is_active": true, "workspace_id": "ws_abc", "created_at": "...", "metadata": {} },
    { "id": "ses_def456", "is_active": true, "workspace_id": "ws_abc", "created_at": "...", "metadata": {} }
  ],
  "total": 24,
  "page": 1,
  "size": 50,
  "pages": 1
}
```

**Panel usage:** Sessions page table.

***

### `POST /v3/workspaces/{workspace_id}/sessions`

Get or create a session. Optionally attaches peers on creation. Returns `200`.

**Request:**

```json
{
  "id": "session-1",
  "peers": {
    "user-123": { "observe_me": true, "observe_others": false },
    "assistant": { "observe_me": true, "observe_others": false }
  }
}
```

**Panel usage:** Implicit — sessions are created when peers first interact.

***

### `DELETE /v3/workspaces/{workspace_id}/sessions/{session_id}`

Delete a session and all its messages (asynchronous, returns `202`).

**Panel usage:** Session list — delete action.

***

### `GET /v3/workspaces/{workspace_id}/sessions/{session_id}/context`

Prompt-ready context bundle: messages + summary + peer representation + peer card.

**Query parameters:**

| Param                  | Type   | Default | Description                                         |
| ---------------------- | ------ | ------- | --------------------------------------------------- |
| `tokens`               | int    | —       | Max token budget (up to 100000)                     |
| `summary`              | bool   | true    | Include session summary                             |
| `peer_target`          | str    | —       | Target peer for representation                      |
| `peer_perspective`     | str    | —       | Perspective peer (must use with `peer_target`)      |
| `search_query`         | str    | —       | Semantic search query for conclusions               |
| `search_top_k`         | int    | —       | Search results to include (1–100)                   |
| `search_max_distance`  | float  | —       | Max distance for semantic search (0–1)              |
| `max_conclusions`      | int    | —       | Max conclusions to include (1–100)                  |
| `include_most_frequent`| bool   | false   | Include most frequent conclusions                   |
| `limit_to_session`     | bool   | false   | Only use messages from this session                 |

**Response:**

```json
{
  "id": "ses_abc123",
  "messages": [
    { "id": "msg_1", "content": "Hello!", "peer_id": "user-123", "session_id": "ses_abc", "created_at": "..." },
    { "id": "msg_2", "content": "Hi! How can I help?", "peer_id": "assistant", "session_id": "ses_abc", "created_at": "..." }
  ],
  "summary": {
    "content": "A greeting exchange.",
    "message_id": "msg_2",
    "summary_type": "short",
    "created_at": "2025-01-15T10:30:00Z",
    "token_count": 45
  },
  "peer_representation": "# user-123\n...",
  "peer_card": ["Name: user-123"]
}
```

**Panel usage:** Session View page — loads all messages and context.

***

## 7. Messages

### `POST /v3/workspaces/{workspace_id}/sessions/{session_id}/messages`

Batch create one or more messages on a session (up to 100).

**Request:**

```json
{
  "messages": [
    {
      "content": "Hey there — can you help me with my math homework?",
      "peer_id": "user-123"
    },
    {
      "content": "Absolutely. Send me your first problem!",
      "peer_id": "assistant"
    }
  ]
}
```

**Response** `201`:

```json
[
  {
    "id": "msg_abc",
    "content": "Hey there — can you help me with my math homework?",
    "peer_id": "user-123",
    "session_id": "ses_abc",
    "workspace_id": "ws_abc",
    "created_at": "2025-05-14T14:30:22Z",
    "metadata": {},
    "token_count": 12
  }
]
```

**Panel usage:** Peer Detail Chat — sending a message.

***

### `POST /v3/workspaces/{workspace_id}/sessions/{session_id}/messages/list`

List messages (paginated, with optional filters).

**Request:**

```json
{
  "filters": {}
}
```

**Query parameters:**

| Param     | Type | Default | Description             |
| --------- | ---- | ------- | ----------------------- |
| `reverse` | bool | false   | Reverse message order   |
| `page`    | int  | 1       | Page number             |
| `size`    | int  | 50      | Page size               |

**Response:**

```json
{
  "items": [
    { "id": "msg_1", "content": "...", "peer_id": "user-123", "session_id": "ses_abc", "created_at": "...", "token_count": 8 },
    { "id": "msg_2", "content": "...", "peer_id": "assistant", "session_id": "ses_abc", "created_at": "...", "token_count": 12 }
  ],
  "total": 18,
  "page": 1,
  "size": 50,
  "pages": 1
}
```

**Panel usage:** Session View — message timeline

## 8. Conclusions

### `POST /v3/workspaces/{workspace_id}/conclusions/list`

List conclusions (paginated, filterable). Ordered by recency unless `reverse` is true.

**Request:**

```json
{
  "filters": {
    "observed": "user-123"
  }
}
```

**Query parameters:**

| Param     | Type | Default | Description           |
| --------- | ---- | ------- | --------------------- |
| `reverse` | bool | false   | Reverse result order  |
| `page`    | int  | 1       | Page number           |
| `size`    | int  | 50      | Page size             |

**Response:**

```json
{
  "items": [
    {
      "id": "con_abc",
      "content": "The user prefers visual explanations over text",
      "observer_id": "honcho",
      "observed_id": "user-123",
      "session_id": "ses_abc",
      "created_at": "2025-05-14T10:00:00Z"
    }
  ],
  "total": 15,
  "page": 1,
  "size": 50,
  "pages": 1
}
```

> `session_id` may be `null` for cross-session conclusions.

**Panel usage:** Conclusions page list.

***

### `POST /v3/workspaces/{workspace_id}/conclusions/query`

Semantic search over conclusions. Returns results ranked by relevance.

**Request:**

```json
{
  "query": "learning preferences",
  "top_k": 10,
  "distance": 0.5,
  "filters": {
    "observed": "user-123"
  }
}
```

**Response:**

```json
{
  "items": [
    {
      "id": "con_abc",
      "content": "The user prefers visual explanations over text",
      "observer_id": "honcho",
      "observed_id": "user-123"
    }
  ]
}
```

**Panel usage:** Conclusions page — semantic search bar.

***

### `DELETE /v3/workspaces/{workspace_id}/conclusions/{conclusion_id}`

Delete a conclusion. Returns `204`.

**Panel usage:** Conclusions page — delete action.
